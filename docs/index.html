Memory Architecture v2 — Foundation Release

Per-project memory
for Claude Code.

A cwd-aware, per-project memory layer built on top of Karpathy's LLM Wiki pattern. Each session loads only what's relevant to the project you're in. Everything else stays quiet.

What breaks at scale

The flat Karpathy pattern works for one project. When you have four:

  • ✗ Every session loads knowledge from all projects. Debugging sideproject while paying for work-api context.
  • ✗ A decision from one codebase bleeds into another. The LLM sees "use Postgres" from a different project and applies it everywhere.
  • ✗ Every session flushes — even sessions that were just ls and pwd. You pay API cost for nothing.
  • ✗ Cost scales with the number of projects in your wiki, not with your actual current work.

How it works

Karpathy original
  • Flat daily log — one file, all projects
  • SessionStart loads full knowledge/index.md
  • Every session flushes to LLM
  • Single MEMORY.md for all context
  • No project isolation
Ramas-Karpathy-Tree
  • daily/<slug>/<date>.md — per project
  • SessionStart loads trunk + active leaf only
  • Pre-filter: trivial sessions skip LLM call ($0)
  • Two-tier MEMORY — global + per-project
  • Strict slug isolation via cwd routing
slug_router.py — longest-prefix cwd match
def resolve_slug(cwd: str, config_path: Path) -> str:
    """Return slug for cwd, or '_global' if no prefix matches."""
    config = json.loads(config_path.read_text())
    routes = config.get("routes", [])
    fallback = config.get("fallback_slug", "_global")

    best_slug, best_len = fallback, 0
    for route in routes:
        for prefix in route.get("cwd_prefixes", []):
            if cwd.startswith(prefix) and len(prefix) > best_len:
                best_len = len(prefix)
                best_slug = route["slug"]

    return best_slug
pre_filter.py — deterministic skip, no LLM cost
def should_flush(transcript: str, char_threshold: int = 1000) -> FilterDecision:
    if not transcript.strip():
        return FilterDecision(False, FilterReason.EMPTY)

    if len(transcript) < char_threshold:
        return FilterDecision(False, FilterReason.TOO_SHORT)

    # all lines match trivial bash / no-op assistant patterns?
    non_trivial = sum(
        1 for line in transcript.splitlines()
        if not any(p.search(line) for p in TRIVIAL_PATTERNS)
    )
    if non_trivial == 0:
        return FilterDecision(False, FilterReason.TRIVIAL_OPS)

    return FilterDecision(True, FilterReason.OK)

Karpathy original vs Ramas-Karpathy-Tree

Dimension Karpathy LLM Wiki Ramas-Karpathy-Tree
Daily logs daily/<date>.md — flat, all projects daily/<slug>/<date>.md new
Session start Loads full knowledge/index.md Trunk + active slug's leaf only new
Project isolation None Strict — slug router prevents leaks new
Pre-filter None — every session costs money Deterministic Python — ~40-60% skip rate new
Flush model Configurable Haiku 4.5 — ~$0.005/flush
Memory structure One MEMORY.md Global (≤20 lines) + per-project (≤30 lines) new
Test coverage Not specified 18 unit tests, TDD from day 1

Cost breakdown

Active developer, 4 projects, ~3-5 sessions/day.

Session flush — Haiku 4.5 (~100/mo, after pre-filter) ~$0.50
Session start context — Sonnet 4.6 (~150/mo) ~$1.50
Plan B: nightly compile — Sonnet 4.6 (coming soon) ~$3.00
Total / month ~$5–15

Naive flat approach with same workload: $30–80/month.

Quick start

Mac, Python 3.12+, uv, Claude Code 1.x. Full guide in README.

1. Clone
git clone https://github.com/sergeyramas/ramas-karpathy-tree \
  ~/.claude-memory-compiler
cd ~/.claude-memory-compiler
2. Install
uv sync
3. Configure projects.json
cp projects.json.example projects.json
# Edit: add your cwd → slug mappings
4. Set env vars (add to ~/.zshrc)
export VAULT_DIR="$HOME/Library/Mobile Documents/iCloud~md~obsidian/Documents/YourVault"
export UV_BIN="$HOME/.local/bin/uv"
5. Register hooks in ~/.claude/settings.json, then run tests
uv run pytest tests/ -v
# 18 passed — you're set.
